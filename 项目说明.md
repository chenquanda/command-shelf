# CommandShelf

CommandShelf 是一个面向个人使用的本地优先常用命令看板。它把 Linux、Docker、Git、SSH 等平时真正会用、但不容易完整记住的命令放在位置稳定的分类中，让用户通过主动翻看和重复接触逐渐形成记忆。

项目坚持轻量、快速、克制：不接入 AI，不提供搜索，不建设命令百科，也不直接执行命令。每条命令除标题、命令正文和用途说明外，还应提供用法、参数说明和参考输出，让用户在执行前知道如何使用以及大致会看到什么。

## 当前内容

- `index.html`：保留的单文件产品原型，用于设计回归和浏览器独立预览。
- `frontend/index.html`：正式 Tauri 桌面前端；已接入真实文档维护、安全拉取、受控提交和普通推送。
- `src-tauri`：Rust 桌面后端、数据契约、原子文件与备份能力、Git 安全同步、测试与 Windows 打包配置。
- `scripts/desktop-smoke.mjs`：通过隔离环境和真实桌面入口执行连接、编辑、拉取、推送及失败保护验收。
- `原型预览.png`：当前 1440×1024 桌面视口下的原型视觉预览。
- `同步原型预览.png`、`同步原型预览-窄屏.png`：GitHub 同步区域在常规与窄屏窗口下的验收截图。
- `design-qa.md`：记录设计参考、同屏对照、修正历史与最终验收结果。
- `docs/product/产品构想.md`：从想法仓库迁移的完整产品构想。
- `docs/ai-project/项目工作台.md`：记录当前阶段、范围、验收标准和恢复入口。

## 查看原型

直接使用浏览器打开项目根目录中的 `index.html` 即可。原型无外部依赖，不需要安装、构建或联网。

## 构建正式桌面应用

开发机需要 Rust、Tauri CLI、MSVC Build Tools 和 WebView2。项目不要求 Node 构建前端；Node 只用于可选的桌面验收脚本。

```powershell
cargo test --manifest-path src-tauri\Cargo.toml --all-targets
cargo clippy --manifest-path src-tauri\Cargo.toml --all-targets -- -D warnings
cargo tauri build --no-bundle
```

## 当前约束

- 根目录原型中的新增、编辑和排序只保存在页面内存中；正式桌面应用的真实数据能力位于 `frontend` 与 `src-tauri`。
- 产品信息结构、同步交互、视觉方向和 Tauri 2 桌面技术方案已经确认；S1～S6 代码、Windows x64 发布候选与非界面自动化验收均已完成，当前等待用户完成双尺寸、键盘和安装/卸载界面验收。
- 正式桌面应用使用独立 Git 仓库中的 `commands.json` 保存命令数据，并通过系统 Git 提供显式拉取与普通推送；浏览器原型不调用本机 Git。
- GitHub 程序仓库为 `git@github.com:chenquanda/command-shelf.git`，个人数据仓库为 `git@github.com:chenquanda/command-shelf-data.git`；应用只依赖用户选择的本地路径，不写死远端仓库名。
- Git for Windows 是启用仓库连接和多电脑同步的前置条件；未安装时应用明确提示安装，不维护第二套无 Git 数据模式。跨电脑验收使用本机两个隔离克隆，不要求第二台物理电脑。
- 从 S5 起，只有不存在前后依赖、文件所有权互斥且可以独立验证的任务，才交给子代理在各自 worktree 并行开发；通过后合并到本地 `main` 并复验，未经用户明确授权不推送远端。
- 后续修改以 `F:\workspace\small-projects\command-shelf` 为唯一项目目录；OpenDesign 临时工程仅保留为历史预览，不再作为主文件维护。
