# CommandShelf

CommandShelf 是一个面向个人使用的 Windows 常用命令看板，用来保存 Linux、Docker、Git、SSH 等容易忘记的命令。每条记录可以包含命令说明、用法、参数说明和参考输出。

应用采用本地优先设计：命令数据保存在用户选择的 Git 仓库中，浏览和编辑不依赖网络；需要在多台电脑间同步时，由用户明确点击“拉取”或“推送”。

## 主要功能

- 按分类浏览、新增和编辑命令。
- 复制命令正文或参考输出。
- 使用拖动或 `Alt+↑`、`Alt+↓` 调整命令顺序。
- 通过系统 Git 快进拉取和普通推送 `commands.json`。
- 在同步失败时保留本地数据，不自动合并冲突或强制覆盖远端。

## 项目结构

- `frontend/index.html`：正式 Tauri 桌面前端。
- `src-tauri`：Rust 后端、数据持久化、备份、Git 同步和 Windows 打包配置。
- `scripts/desktop-smoke.mjs`：真实桌面入口的自动验收脚本。
- `scripts/release-candidate.ps1`：测试、静态检查和 Windows 安装包构建脚本。
- `docs/安装与使用说明.md`：安装、数据仓库配置和日常使用说明。
- `release`：随源码仓库保存的 Windows 安装包。

## 开发与验证

开发机需要 Rust、Tauri CLI、MSVC Build Tools、WebView2 和 Git for Windows。前端不需要 Node 构建；Node 仅供桌面验收脚本使用。

```powershell
cargo fmt --manifest-path src-tauri\Cargo.toml --check
cargo test --manifest-path src-tauri\Cargo.toml --all-targets
cargo clippy --manifest-path src-tauri\Cargo.toml --all-targets -- -D warnings
powershell -ExecutionPolicy Bypass -File scripts\release-candidate.ps1
```

发布脚本会在项目 `.local` 下重新创建编译缓存、运行期临时目录、发布产物和验证证据。这些内容均被 Git 忽略，可以在构建完成后清理。

## 数据与同步约束

- 数据仓库只需要维护一个 `commands.json`，当前格式版本为 `schemaVersion: 1`。
- 应用只管理 `commands.json`，不会暂存或提交仓库中的其他文件。
- GitHub 程序仓库：`git@github.com:chenquanda/command-shelf.git`。
- GitHub 数据仓库：`git@github.com:chenquanda/command-shelf-data.git`。
- 应用不内置 GitHub 登录、Token 管理、后台同步或冲突解决界面。
- 卸载应用不会删除用户选择的数据仓库。
