## 开始前

Codex 会打开免安装主程序，并准备一个空的本地 Git 测试仓库。首次进入后：

1. 点击左侧“仓库”。
2. 输入交付消息中的“测试仓库路径”并连接。
3. 新建一个“验收”分类，再添加三条简单命令，例如 `git status`、`git log -1`、`git branch`。
4. 等界面显示“本地有修改”后开始下面的检查。

## 需要手工完成的界面检查

### 1. 两种窗口尺寸

分别在约 1440×1024 和 1024×768 的窗口下检查：

- 页面没有横向滚动条、遮挡或内容重叠。
- 仓库、拉取、推送、添加分类和添加命令均可看到并可操作。
- 命令说明、用法、参数说明和参考输出仍然清晰可读。

### 2. 键盘操作

- 打开“仓库”对话框，确认 Tab 与 Shift+Tab 不会跑到背景页面，Escape 能关闭并回到“仓库”按钮。
- 打开“添加命令”或“编辑命令”抽屉，确认焦点先进入标题；Tab 与 Shift+Tab 在抽屉内循环；Escape 关闭后回到原按钮。
- 在第二条命令的排序手柄上按 Alt+↑，确认它上移且焦点仍在同一命令；再按 Alt+↓ 恢复。
- 在第一条按 Alt+↑、最后一条按 Alt+↓，确认顺序不变并出现明确提示。
- 编辑一条命令并保存，确认保存完成后焦点回到该命令的编辑按钮。

### 3. 安装与卸载

1. 退出当前免安装主程序。
2. 运行 NSIS 安装包，完成当前用户安装。
3. 从开始菜单启动 CommandShelf，重新连接测试仓库，确认三条命令仍在。
4. 关闭应用，在 Windows“已安装的应用”中卸载 CommandShelf。
5. 确认应用入口已移除，同时测试仓库目录及其中的 `commands.json` 没有被删除。
