# CommandShelf 安装与使用说明

## 安装

Windows x64 安装包位于：

`F:\workspace\small-projects\command-shelf\release\CommandShelf_0.1.0_x64-setup.exe`

安装包 SHA-256：

`072E4204A3FAF518C7725CE610AA0DE963FF28B23F35D27C9302970FBDDAFC5D`

运行安装包后，CommandShelf 会安装到当前 Windows 用户，并在开始菜单创建入口。连接数据仓库前需要安装 Git for Windows；如果系统找不到 Git，应用会给出安装提示。

Codex 命令草稿属于可选功能。需要使用时，请确保本机 Codex CLI 已安装、已完成登录，并且系统终端能够执行：

```powershell
codex --version
```

如果这条命令不可用，仍然可以正常浏览、编辑和同步个人命令，只是“问 Codex”对话框会提示检查安装。

## 准备数据仓库

首次使用时，先在系统终端克隆个人数据仓库，例如：

```powershell
git clone git@github.com:chenquanda/command-shelf-data.git F:\CommandShelfData
```

数据仓库需要有可用的 `origin`、当前分支和上游分支。然后在 CommandShelf 左侧点击仓库名称，填写本地仓库根目录，例如 `F:\CommandShelfData`。

如果仓库中还没有 `commands.json`，应用会创建空数据文件。首次推送前还需要确保系统 Git 已配置提交身份：

```powershell
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

## 日常使用

建议在一台电脑上按以下顺序操作：

1. 打开应用后点击“拉取”。
2. 新增、编辑或排序命令；修改会立即保存在本地 `commands.json`。
3. 状态显示“本地有修改”后点击“推送”。
4. 换到另一台电脑时，先拉取再继续修改。

应用只执行快进拉取和普通推送。如果本地存在未推送内容、远端领先或分支已经分叉，应用会停止同步并保留本地数据，需要用户在系统终端处理 Git 状态。

## 使用 Codex 生成命令草稿

1. 先进入草稿最终应归属的分类，再点击顶部“问 Codex”。
2. 用自然语言描述目标、系统环境和必要限制，然后点击“生成命令”。
3. 生成成功后，完整命令卡片会作为“临时结果”显示在当前分类顶部；此时不会写入 `commands.json`，也不会改变 Git 状态。
4. 检查命令、说明、用法、参数和参考输出。确认需要保留时，点击“保存到当前分类”，命令才会写入当前分类第一条。
5. 如果 Codex 连续两次没有返回合法结构，对话框会展示第二次原文；按提示人工补齐字段后，可以先生成临时卡片再决定是否保存。

CommandShelf 只请求和展示命令，绝不会执行、试运行或验证生成的命令。未点击保存就退出应用或开始下一次生成，不会把旧临时结果写入数据仓库；实际使用前仍应自行检查命令是否适合当前环境。

## 卸载与数据

可以在 Windows“已安装的应用”中卸载 CommandShelf。卸载会移除程序目录、开始菜单入口和安装记录，但不会删除用户选择的数据仓库及其中的 `commands.json`。

当前安装包已经通过 Rust 46 项测试、Clippy 零警告、格式检查、前端语法检查和 Windows x64 NSIS 构建。安装包与独立 EXE 来自同一次发布候选构建。

请在本次新安装包上手工复验：安装与启动、仓库连接、问 Codex、临时结果不落盘、保存后位于分类首位、重启后仍存在、键盘焦点、最小窗口，以及卸载后数据仓库仍保留。
