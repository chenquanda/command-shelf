# CommandShelf 安装与使用说明

## 安装

Windows x64 安装包位于：

`release\CommandShelf_0.1.0_x64-setup.exe`

安装包 SHA-256：

`1F03448C493AD746DA1648894970C46C65F928F577B37ACEF022F2C903D2372D`

运行安装包后，CommandShelf 会安装到当前 Windows 用户并创建开始菜单入口。应用使用系统 WebView2 显示界面；连接数据仓库前需要安装 Git for Windows。

Codex CLI 是可选依赖。只有使用“问 Codex”时，才需要确保系统终端能够执行：

```powershell
codex --version
```

Codex CLI 缺失不会影响手工管理命令、临时收集或 Git 同步。

## 准备数据仓库

先在系统终端克隆个人数据仓库，例如：

```powershell
git clone git@github.com:C-Q-D/command-shelf-data.git F:\CommandShelfData
```

数据仓库需要具有 `origin`、当前分支和可用上游。首次推送前还需要配置 Git 提交身份：

```powershell
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

首次启动应用后，点击左侧仓库名称，选择数据仓库根目录。应用会读取并校验：

- `commands.json`：正式命令数据；缺失时在连接仓库过程中创建空文档。
- `inbox.json`：临时收集数据；缺失时在首次进入“临时收集”时创建空文档。

无效文件不会被空文档覆盖。应用只管理这两个文件，不会提交数据仓库中的其他内容。

## 管理正式命令

- 在左侧选择分类，使用“添加命令”保存说明、命令正文、用法、参数和参考输出。
- 可以编辑已有命令，也可以拖动或使用 `Alt+↑`、`Alt+↓` 调整顺序。
- 复制正式命令正文后，复制次数会原位增加并在后台保存；复制参考输出不计数。
- 点击“按次数排序”才会按复制次数降序保存，同次数记录保持原相对顺序。
- 连续复制会串行保存；新增、编辑、排序或同步会等待复制次数先落盘。

## 使用临时收集

“临时收集”固定显示在分类目录之前，不属于用户分类。

- 输入普通文字、链接或混合内容后，点击“记下来”或按 `Ctrl + Enter` 保存。
- 新记录显示在时间流顶部，并按“今天”“昨天”或具体日期分组。
- 点击编辑按钮可以行内修改；取消或按 `Escape` 不会写入文件。
- 点击删除按钮后必须确认；保存失败时记录、顺序和焦点都会恢复。
- 所有修改立即安全写入 `inbox.json`，但不会自动访问网络。

## 使用 Codex 生成命令草稿

1. 进入草稿最终应归属的分类，点击“问 Codex”。
2. 描述目标、系统环境和限制，然后生成命令。
3. 合法结果会作为当前分类顶部的临时卡片展示，不会立即写入 `commands.json`。
4. 检查说明、用法、参数、参考输出和风险提示。
5. 点击“保存到当前分类”后，草稿才会成为正式命令。

应用只请求和展示命令，绝不会执行、试运行或验证生成的命令。连续两次无法获得合法结构时，会保留第二次原文供用户人工补齐。

## 拉取与推送

推荐使用顺序：

1. 打开应用后点击“拉取”。
2. 新增、编辑、排序或删除内容。
3. 状态显示“本地有修改”后点击“推送”。
4. 换到另一台电脑时先拉取，再继续修改。

同步规则：

- 网络操作只在用户点击后执行，不在启动、退出或后台定时运行。
- 拉取只接受快进，不覆盖未提交或未推送的本地内容。
- 推送只暂存 `commands.json` 和 `inbox.json`，不使用强推。
- 分支分叉、认证失败、远端领先、网络中断或超时都会安全停止并保留本地数据。
- 同步期间会暂时禁用数据修改；浏览和复制参考输出仍可使用。

## 升级与卸载

运行新版安装包即可升级。数据仓库位于用户选择的独立目录，不随程序升级或卸载而删除。

可以在 Windows“已安装的应用”中卸载 CommandShelf。卸载会移除程序目录、开始菜单入口和安装记录，但会保留：

- 用户的数据仓库及其中的 `commands.json`、`inbox.json`。
- 当前机器保存的数据仓库路径配置和本地备份，除非用户另行手工清理。

## 当前安装包验收状态

当前安装包已经通过 Rust 63 项测试、Clippy 零警告、格式检查、前端语法、复制无刷新回归、临时收集专项回归和 Windows x64 NSIS 构建。独立 EXE 已通过窗口响应与 WebView2 启动冒烟。

建议在实际安装后手工检查：

1. 安装、开始菜单启动和升级覆盖。
2. 仓库连接，以及拉取或推送期间窗口可以正常拖动。
3. 临时收集新增、编辑、取消、删除和重启恢复。
4. 命令正文复制计数、参考输出不计数和按次数排序。
5. “问 Codex”、临时结果不落盘和显式保存。
6. 最小窗口、键盘焦点和长内容滚动。
7. 卸载后数据仓库仍然保留。

## 开发机重新生成安装包

提交当前源码后执行：

```powershell
pwsh -ExecutionPolicy Bypass -File scripts\生成安装包.ps1
```

脚本会更新 `release` 中的安装包及本说明中的 SHA-256，但不会自动安装、提交 Git 或推送远端。
